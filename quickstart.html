SparkNLP - Quick Start

Last updated: Sep 13th, 2017

Concepts

Since SparkNLP runs on top of Spark, make sure you achieve a working spark-shell or pyspark setup, and you understand how to include a library dependency into your project. Please refer to the spark documentation to get started.

  • Python
  • Scala

Run spark-shell --jars /path/to/spark-nlp.jar along with any spark options you might want to use.

run pyspark --jars /path/to/spark-nlp.jar along with desired spark launcher options.

Concepts

One of Spark modules, called ML, provides a set of Machine Learning applications, and it's logic consists of two main components: Estimators and Transformers. The first, have a method called fit() which secures and trains a piece of data to such application, and a Transformer, applies changes to the result of an estimators plan. These components have been embedded to be applicable to Spark NLP. Pipelines are a mechanism that allows multiple transforms within a single process workflow of transformers, allowing multiple chained transformations along a Machine Learning task. Refer to Spark ML docs for further information and features.

Document (Getting data into)

In SparkNLP, a Document is a raw-data container, which comes along with related information of such entry. Every line of text, or piece of raw data, shall be contained within a document in order to be used by SparkNLP tasks.

A document is made of:
  • text: raw-data.
  • id: an optional reference value.
  • metadata: optional information in the shape of a string dictionary.

A Document can be created by utilizing a transformer called DocumentAssembler. This will take the elements of a Document in the shape of Dataset columns.

Functions

  • setInputCol(col_name)
  • setIdColName(col_name)
  • setMetadataColName(col_name)
  • setOutputCol(col_name)

Example

from sparknlp.base import DocumentAssembler
data = spark. \
  read. \
  parquet("some_data.parquet")
document_assembler = DocumentAssembler() \
  .setInputCol("text") \
  .setOutputCol("document")
document_assembler.transform(data).show()

Annotation (NLP Content)

An annotation is the basic form of the result of an NLP operation. It is made of:

  • annotatorType: which annotator generated this annotation
  • begin: the begin of the matched content relative to raw-text
  • end: the end of the matched content relative to raw-text
  • metadata: content of matched result and additional information

This object is automatically generated by annotators. No manual work is required. But it must be understood in order to extract information out of it. API functions are provided in order to interact with annotations.

Annotators

Annotators are the spearhead of NLP functionalities in SparkNLP. There are two types of annotators:

  • Annotator Approach: Are those who represent a Spark ML Estimator and require a training stage. They have a function called fit(data) which trains a model based on some data. They produce the second type of annotator which is an annotator model
  • Annotator Model: They are spark models or transformers, meaning they have a transform(data) function which take a dataframe and add to it a column with the result of this annotation.

Both annotator types can be included in a Pipeline and will automatically go through all stages in the provided order and transform the data accordingly.

Functions

  • setDocumentCol(column_name): Takes the column name that refers to the Document column generated before
  • setInputCols(column_names): Takes a list of column names of annotator outputs required by this annotator
  • setOutputCol(column_name): Defines the name of the column containing the result of this annotator. Use this name as an input for other annotators requiring this one.

Example

document_assembler = DocumentAssembler() \
  .setInputCol("text") \
  .setOutputCol("document")
tokenizer = RegexTokenizer()\
  .setInputCols(["document"]) \
  .setOutputCol("token")
assembled = document_assembler.transform(data)
tokenized = tokenizer.transform(assembled)
tokenized.transform(data).show()

Annotator types

Every annotator has a type. Those annotators that share a type, can be used interchangeably, meaning you could you use any of them when needed. For example, when a token type annotator is required by another annotator, such as a sentiment analysis annotator, you can either provide a normalized token or a lemma, as both are of type token.

Using Pipelines

document_assembler = DocumentAssembler() \
            .setInputCol("text")

sentence_detector = SentenceDetector() \
    .setInputCols(["document"]) \
    .setOutputCol("sentence")

tokenizer = RegexTokenizer() \
            .setOutputCol("token")

spell_checker = SpellChecker() \
            .setInputCols(["token"]) \
            .setOutputCol("spell")

sentiment_detector = ViveknSentimentDetector() \
    .setInputCols(["spell", "sentence"]) \
    .setOutputCol("sentiment") \
    .setPositiveSource("vivekn/positive") \
    .setNegativeSource("vivekn/negative") \
    .setPruneCorpus(False)

pipeline = Pipeline(stages=[
    document_assembler,
    sentence_detector,
    tokenizer,
    spell_checker,
    sentiment_detector
])

sentiment_data = pipeline.fit(data).transform(data)
